# Git Commit & Push Skill

Commit and push changes following Hive Mind patterns.

## Stage changes
```bash
git add -A
git add <file>
```

## Commit
```bash
git commit -m "type: description"
```

## Push
```bash
git push origin <branch>
git push -u origin <branch>
```

## Conventional Commits
- `feat:` - New feature
- `fix:` - Bug fix
- `chore:` - Maintenance
- `docs:` - Documentation
- `refactor:` - Code refactoring
- `test:` - Tests
- `perf:` - Performance

## Rules

1. One commit = one logical change
2. Use conventional commits
3. Never force push to main
4. Push feature branches only
